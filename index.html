操作系统如何管理内存操作系统提供一种机制以将虚拟地址和物理地址映射起来，程序访问虚拟地址的时候，会由操作系统转换到不同的物理地址，以避免内存冲突。在程序中使用的内存地址叫做虚拟内存地址(virtual memory address)，物理内存地址(physical memory address)，进程持有的虚拟地址会通过CPU芯片中的内存管理单元(MMU)来映射到物理地址。 内存分段内存分段将整个内存空间划分为多个连续的段，每个段包含一个或多个进程或数据结构，不同的段之间可以有不同的访问权限和保护机制。 内存分段机制下，虚拟地址由两部分组成，即段选择因子和段内偏移量。 段选择因子包括段号和其他标志位，段号是段表的索引，段表中保存了段的基地址、界限、特权等级等。在端内偏移量合法的情况下，将段基地址加上段内偏移量得到物理内存地址。 内存碎片 内存分段会根据实际需求分配内存，有多少需求就分配多大的段，所以不会出现外部内存碎片，但由于每个段的长度不固定，所以内存可能被分成多个小的、不连续的段。 为了解决内存碎片的问题，可以考虑内存交换，即将两个碎片之间的程序内存占用写到硬盘上，释放内存使 ...

创建型模式这些设计模式提供了一种在创建对象的同时隐藏创建逻辑的方式，而不是使用 new 运算符直接实例化对象。这使得程序在判断针对某个给定实例需要创建哪些对象时更加灵活。 单例模式单例模式设计到一个单一的类，该类负责创建自己的对象，同时确保只有单个对象被创建。通过这个类可以直接访问其唯一的对象，并且不需要实例化该类的对象。 注意： 单例类只能有一个实例。 单例类必须创建自己的唯一实例。 单例类必须给其他所有对象提供这一实例。 工厂模式工厂模式定义创建对象的接口，让子类自己决定实例化哪个工厂类，工厂模式使其创建过程延迟到子类进行。工厂模式会增加系统的复杂度，但其有着良好的屏蔽性和扩展性。 抽象工厂模式可以简单理解为工厂的工厂。 建造者模式建造者模式使用多个简单的对象一步一步构建出一个复杂的对象。与工厂模式相比，建造者模式更加关注顺序。 原型模式原型模式用于创建重复的对象，同时又能保证性能。 原型模式实现了一个接口，用于创建当前对象的克隆，如果直接创建对象的代价比较大时，可以使用这种模式。 结构型模式这些设计模式关注类和对象的组合。继承的概念被用来组合接口 ...
