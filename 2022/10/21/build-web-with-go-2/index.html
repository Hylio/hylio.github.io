结构体

  1. Go支持只提供类型，而不写字段名的方式。即匿名字段，也称为嵌入字段。当匿名字段是一个结构体时，那么该匿名结构体的所有字段都被隐式地引入当前定义中的结构体。

  2. 当内部结构体和外部结构体拥有同样名称的字段时，优先访问外部的。如果想访问匿名字段中的同名字段，可以通过先调用匿名字段名来访问。

  3. 方法可以作用在任何的自定义类型、内置类型、struct等各种类型上面。

  4. 如果要对结构体的内容进行更改，则需要将指针（而不是结构体自身）作为方法的接收者。如果不传递指针，那么方法接收到的实际上是结构体的一个copy，对其的更改并不能真正作用在原结构体上。

  5. Go可以自动识别需要的是指针or指针对应的值，也就是说：如果一个method的receiver是*T,你可以在一个T类型的实例变量V上面调用这个method，而不需要&V去调用这个method；同样的，如果一个method的receiver是T，你可以在一个*T类型的变量P上面调用这个method，而不需要 *P去调用这个method。

  6. 与字段相同，method也能够继承。如果匿名字段实现了一个method，那么包含这个字段的struct也能够调用这个method。如果想要重写method，则直接为当前struct定义一个method即可。

接口

  1. 接口是一组方法的集合，如果某个对象能够实现该接口的所有方法，那么此对象就实现了此接口。接口可以被任意多个对象实现，一个对象也可以实现任意多个接口。

  2. 如果定义一个interface的变量，那么这个变量里面可以存储实现了这个interface的任意类型的对象。如果定义一个interface类型的slice，这个slice也可以被赋予实现了该interface的任意对象，甚至可以在同一个slice中存储不同类的对象，只要他们实现了这个slice对应的interface。

  3. 空interface(interface{})不包含任何method，正因如此，所有的类型都实现了空interface。空interface对描述起不到任何作用，但它使得我们可以存储任意类型的数值。一个函数如果把interface{}作为参数，那么说明它可以接收任意类型的值作为参数。一个函数如果返回interface{}作为返回值，那么也就可以返回任意类型的值。

  4. 可以有两种方法来确定一个interface变量中存储的是何种类型的对象。

    • 断言

    • 通过switch来判断，注意element.(type)只能在switch语句中使用。

  5. 与匿名字段相似，如果一个interface1作为interface2的嵌入手段，那么interface2隐式的包含了interface1中所有method。例如在源码中，heap接口实际上就是包含了sort接口。

    • 转化为reflect对象后，可以从中获取值。

    • 反射的字段可以修改，如果想要修改，那必须在一开始传入指针。

并发

  1. goroutine是Go并行设计的核心。goroutine是比线程更小的协程，Go语言底层实现了这些goroutine之间的内存共享，执行起来只需要极少的栈内存。所以goroutine比thread更易用、更高效、更轻便。

  2. 为了做好goroutine之间的贡献内存，显然需要进行goroutine的数据通信。Go提供了一个很好的通信机制：channel。channel可以与unix shell中的双向管道类比，可以通过channel发送或者接收值。在定义一个channel时，必须定义发送到channel的值的类型。必须使用make创建channel。

    channel通过操作符<-来接收和发送数据。

    在默认情况下，channel发送和接收数据都是阻塞的，除非另一端已经准备好了。这使得goroutine在同步时更加简单，而不需要显式的lock。所谓的阻塞就是指，读取（value := <-ch）将会被阻塞，直到有数据接收。其次，任何发送（ch<-5）也将会被阻塞，直到数据被读出。

  3. 在Go中，也可以指定channel的缓冲大小，即channel可以存储多少元素。当创造一个大小为len的channel时，前len个元素可以无阻塞地写入，而写入第len+1个元素时，将会被阻塞，直到其他goroutine从channel中读取元素。当给定的len=0时，实际上相当于定义一个无缓冲阻塞读写的channel。

  4. 可以通过range语句，像操作slice或map那样操作channel。

    可以通过close来关闭channel，关闭channel之后，channel便无法再发送任何数据了。可以通过语法v, ok := <- ch来检测channel是否被关闭。如果ok为false，则说明channel已经被关闭。

  5. 当存在多个channel时，可以使用关键字select来监听channel上的数据流动。select默认是阻塞的，只有当监听的channel中有发送或者接收可以进行时，才会运行。如果同时有多个channel准备好，那么会随机选择一个正在执行的channel。

    类似switch语句，select中也有default语句，当监听的channel都没有准备好的时候，默认执行default语句。

  6. 有时候会出现goroutine阻塞的情况，为了避免整个程序进入阻塞，可以利用select来设置超时。

  7. runtime包中一些常用的处理goroutine的函数：

    • Goexit

      退出当前执行的goroutine，但defer函数继续调用。

    • Gosched

      让出goroutine的执行权限，等待其他任务运行，在下次某个时候从该位置恢复执行。

    • NumCPU

      返回CPU核数量。

    • NumGoroutine

      返回正在执行和排队的任务总数。

    • GOMAXPROCS

      设置可以并行计算的CPU核的数目最大值，并返回之前设置的值。
